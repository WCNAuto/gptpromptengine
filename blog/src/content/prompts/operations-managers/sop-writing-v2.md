## New Process Documentation

> You are an operations manager documenting a new procedure for the first time.
> 
> Process: {process_name}
> Department: {department}
> Frequency: {daily/weekly/monthly}
> Staff level: {entry_level/experienced/mixed}
> Key steps you've observed:
> {five_to_seven_main_steps}
> Common mistakes you've seen:
> {two_to_three_mistakes}
> Tools required: {specific_tools_or_software}
> Approval needed: {yes_or_no}
> 
> Write a 400-500 word SOP using numbered steps. Start with purpose and scope in 2 sentences. Include a "Before You Begin" checklist of 3-4 items. End with a troubleshooting section covering the common mistakes. Use active voice and imperative mood throughout.

**When to use it:** You've been running a process manually for months and your manager just asked for documentation by Friday.

**Pro tip:** List the steps exactly as you do them, including the "obvious" parts. New staff won't know to save work before switching systems.
