## Template structure

### Portfolio Projects Database

- **Project Name** (Title): "Shopify checkout redesign", "B2B dashboard mobile optimization". The name you'll use in interviews — keep it client-safe if under NDA.

- **Client/Company** (Text): "Shopify", "Stealth fintech startup", "Personal project". Required for tracking NDA status and approval needs.

- **Project Type** (Select): Case study / Side project / Concept work / Course project. Affects which approval process applies and interview relevance weighting.

- **Portfolio Status** (Select): Not started / In progress / Client review / Ready for portfolio / Published. Change to Client review the moment you send case study draft for approval. Only mark Ready when you have written client sign-off.

- **Case Study Progress** (Select): Research documented / Wireframes captured / Visual design saved / Copy written / Published live. Drives the Copy Needed and Visuals Needed filtered views.

- **Interview Readiness** (Select): Not suitable / Needs work / Interview ready / Showcase piece. Only mark Interview ready when you can present it confidently in 3 minutes without preparation.

- **Client Approval Needed** (Checkbox): Check if this requires legal/client review before publication. Drives the Blocked Projects view for follow-up.

- **Target Completion** (Date): When you need this case study finished for applications or interviews. Auto-sorts the Pipeline view by urgency.

- **Story Angle** (Text): "Mobile-first redesign that increased conversion 23%", "0-1 design system for series A startup". Your 15-second pitch for this project.

- **Assets Status** (Select): Missing / Partial / Complete. Mark Complete only when you have high-res visuals, process shots, and outcome metrics saved locally.

- **Notes** (Text): Approval contact names, missing research data, visual gaps. What you need to remember when you return to this project.

## How to use it week by week

1. **Sunday planning**: Open the Pipeline view sorted by Target Completion date. Identify the 2-3 projects with nearest deadlines that aren't marked Interview ready.

2. **Monday work session**: For each project you're advancing this week, update Case Study Progress to reflect current state. If you're sending anything for client review, flip Portfolio Status to Client review and check Client Approval Needed.

3. **Wednesday check-in**: Review the Blocked Projects view (Client Approval Needed = checked, Portfolio Status = Client review). Send follow-up emails to any approvals pending over 5 days.

4. **Friday wrap-up**: Update Assets Status for any projects where you captured new visuals or gathered additional research. Change Interview Readiness for any projects you could now present confidently without preparation.

5. **Before any interview**: Filter Portfolio Status to Interview ready, sort by Interview Readiness. Your Showcase pieces go in the presentation, Interview ready pieces become your backup examples.

## Frequently Asked Questions

### What if a project needs client approval but they're not responding?

Set Portfolio Status to Blocked and use the Notes field to track your follow-up attempts with dates. The Blocked Projects view will keep these visible during your weekly reviews so nothing falls through the cracks.

### Should I track personal projects differently than client work?

Use the same database but set Project Type to Personal project or Concept work. These skip the Client Approval Needed workflow but still need Assets Status and Story Angle defined before they're Interview ready.

### How do I handle projects under strict NDAs?

Keep them in the tracker but use generic names in Project Name ("E-commerce checkout optimization") and mark Client Approval Needed. Use the Notes field to track what you can/cannot show publicly so you don't accidentally include restricted details.
