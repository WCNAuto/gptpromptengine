## Template structure

### Candidates Database
- **Candidate Name (Title)**: Full name as written on application. Leave empty and the record becomes invisible in main pipeline view.
- **Position Applied (Select)**: Software Engineer / Marketing Manager / Sales Rep / HR Coordinator. Must match exactly with your open roles or candidate appears in "Unassigned" filtered view.
- **Application Date (Date)**: Date application received, not date candidate was added to system. Drives the "Applications This Week" view for weekly reporting.
- **Current Stage (Select)**: Application Review / Phone Screen / Technical Interview / Final Interview / Reference Check / Offer Extended / Hired / Rejected. Update immediately after each interaction to keep pipeline accurate.
- **Hiring Manager (Person)**: Assigned decision maker, not recruiter. Required field - candidates without assigned managers appear in "Needs Assignment" view.
- **Next Action Date (Date)**: When the next interview, call, or decision is scheduled. Overdue items automatically appear in "Action Required" view.
- **Interview Notes (Rich Text)**: Feedback from each interview round. Include interviewer name and specific concerns - this feeds into final hiring decisions.
- **Salary Expectation (Number)**: Candidate's stated expectation in annual salary. Leave blank until discussed in phone screen.

### Positions Database
- **Role Title (Title)**: Exact job title as posted. Must match Position Applied options in Candidates database.
- **Department (Select)**: Engineering / Marketing / Sales / Operations. Groups candidates by team in department view.
- **Hiring Manager (Person)**: Final decision maker for this role. Auto-populates when candidates select this position.
- **Target Start Date (Date)**: When role needs to be filled. Drives urgency scoring in candidate views.
- **Status (Select)**: Actively Hiring / On Hold / Filled / Cancelled. Only "Actively Hiring" positions show in main recruitment dashboard.

## How to use it week by week

1. **Monday morning**: Open the "Action Required" view and contact every candidate with overdue Next Action Dates. Update Current Stage for any candidates who completed interviews last week.

2. **When new applications arrive**: Create new candidate record, set Application Date to today, assign Position Applied and Hiring Manager immediately. Schedule phone screen within 48 hours and set Next Action Date.

3. **After each interview**: Open candidate record, add interview feedback to Notes field, update Current Stage, and set Next Action Date for follow-up or next round. Never leave a candidate without a next action.

4. **Thursday afternoon**: Generate weekly report by filtering "Applications This Week" view and copying candidate list into your hiring manager update email.

5. **Friday end-of-day**: Review "Needs Assignment" and "Unassigned" views to catch any candidates who slipped through the process.

## Frequently Asked Questions

### How do I handle candidates applying for multiple positions?
Create separate candidate records for each position - the same person can appear twice with different Position Applied values. This prevents confusion when different hiring managers evaluate the same candidate for different roles.

### What happens when I need to add custom interview stages?
Edit the Current Stage select field to add your specific stages like "Culture Fit Interview" or "CEO Review". Keep stages in chronological order and update the filtered views to include your new stages in the appropriate pipeline sections.

### Can I track group interviews or panel sessions?
Use the Interview Notes field to log all panel member feedback in one entry. Include each interviewer's name and their specific recommendation. Set Next Action Date for when the panel decision is due, not individual follow-ups.
