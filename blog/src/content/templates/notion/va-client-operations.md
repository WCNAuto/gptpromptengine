## Template structure

### Client Operations Database
- **Client Name (Select)**: Client A / Client B / Client C / Client D / Client E. Set this first for every new task — it drives the client-specific filtered views and weekly reports.
- **Task Title (Text)**: "Q3 blog post series outline" or "Weekly social media calendar". Be specific enough that the client recognises it in your status report.
- **Deliverable Type (Select)**: Content / Design / Admin / Research / Strategy. Used to colour-code the timeline view and group similar work in reports.
- **Status (Select)**: Not started / In progress / Submitted / Approved / Delivered. Change to Submitted only when you've actually sent it to the client. Change to Approved only when you have written confirmation.
- **Assigned Date (Date)**: When the client briefed you. Leave empty and the task won't appear in your weekly capacity planning view.
- **Due Date (Date)**: Client's expected deadline. Missing dates break the Overdue Items view that you check every Monday.
- **Submitted Date (Date)**: When you delivered it to the client. Auto-calculates turnaround time in the reporting view.
- **Approval Date (Date)**: When client confirmed it's approved. Drives the "Pending Approval" aging report you send every Friday.
- **Priority (Select)**: High / Medium / Low. High items appear first in every filtered view.
- **Client Notes (Text)**: Specific feedback or requirements from the brief. Copy-paste directly from client emails.

### Client Contact Hub
- **Client Name (Text)**: Matches the select options in the main database exactly.
- **Primary Contact (Text)**: The person who briefs you and approves work.
- **Approval Contact (Text)**: If different from primary contact. Prevents delays when you submit to the wrong person.
- **Weekly Report Day (Select)**: Monday / Tuesday / Wednesday / Thursday / Friday. When they expect their status update.
- **Communication Preference (Select)**: Email / Slack / Project tool. Where to send reports and deliverables.

## How to use it week by week

1. **Every Monday morning**: Open the Overdue Items view. Chase any tasks still showing "Submitted" from last week. Update any that have been approved over the weekend.

2. **When a new brief arrives**: Create a new task, set Client Name and Assigned Date immediately. Set Due Date based on client's timeline. Set Priority to High if it impacts other work.

3. **When you submit any deliverable**: Change Status to Submitted and fill in the Submitted Date. This starts the approval timer that shows in your Friday reports.

4. **Every Friday afternoon**: Filter by each client's name, copy the active tasks into your status report template. The Submitted Date auto-calculates how many days each approval has been pending.

5. **When clients approve work**: Update Status to Approved and fill in Approval Date. This moves the task out of your pending queue and into the completed work log.

## Frequently Asked Questions

### What happens if I don't fill in the Submitted Date?
The pending approval aging calculation breaks. Your Friday reports won't show how long each approval has been outstanding, making it harder to identify which clients need follow-up.

### Can I track billable hours in this template?
No. This template focuses on deliverable status and client communication. Add a separate Time Tracking database and relate it to the Task Title if you need detailed billing records.

### How do I handle recurring tasks like weekly reports?
Create one task per instance: "Weekly report - Week of Jan 6", "Weekly report - Week of Jan 13". This gives you proper completion tracking and prevents confusion about which week's report is overdue.
